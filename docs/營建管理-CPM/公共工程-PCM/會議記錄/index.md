# 會議記錄

在營建管理中，會議紀錄不單是溝通摘要，更是「契約文件（Contract Documents）」的延伸。無論是每週工務會議、施工協調會還是估驗審查會，每一項決議都可能涉及工期展延、成本變更或責任歸屬。其紀錄在法律實務中常被視為事實證據（Contemporaneous Records）。

本系統的「會議紀錄」功能針對營建產業的高風險特性設計，將會議決議與後續的執行單據（派工、RFI）形成不可竄改的邏輯鏈條。這在面對工程展延（EOT）、工期索賠、或是工項變更追加（Variation Order）時，能提供具備時間戳記與權責簽核的證據力，透過數位化結構將會議內容拆解為可量化、可追蹤、可稽核的行動項目。

***

工程涉及土建、機電、裝修、景觀等多專業界面，跨專業的界面衝突（Interface Conflict）是導致進度落後的主因。因此系統採納結構化的議題分類（會議議題-待辦事項）：

{% tabs %}
{% tab title="會議議題 (Agenda Items)" %}
專業分工的節點，使用者可視當下討論焦點，自行定義並掛載標籤（如：#變更設計、#進度檢討、#界面協調）。 這確保了複雜的會議紀錄能被有效分類，方便日後針對特定技術問題進行檢索與溯源。
{% endtab %}

{% tab title="待辦事項  (Action Items)" %}
執行責任的最小單元。每個議題下衍生的待辦事項是系統的靈魂。它將會議共識具體化為「誰、在何時、需完成什麼」。針對如「鋼筋綁紮與機電配管」或「防水施工與貼磚作業」等銜接議題進行專項記錄。將會議中的界面釐清結論，直接轉化為待辦事項，避免了因工序混亂導致的重工（Rework）。
{% endtab %}
{% endtabs %}

本系統最具產業競爭力的特點，在於待辦事項與工程核心單據的「自動化鏈接」。這打破了辦公室文書與現場執行之間的資訊斷層：

!!! info
    #### 功能總結
    
    1. 確保履約證據的完整性 工程爭議往往發生在完工數年後。本系統將會議紀錄與後續的派工、釋疑歷程完整串聯。這種「凡走過必留下痕跡」的數據結構，為專案提供了強大的歷史稽核能力，防止因人員更迭而導致的資訊斷層。
    2. 強化多方協作的透明度 在涉及業主、監造、營造及多個協力廠商的複雜環境下，系統的待辦事項能即時同步給相關責任人。這種透明的追蹤機制能顯著降低溝通誤差，縮短從「發現問題」到「問題解決」的週期。
    3. 待辦事項具備狀態追蹤功能（如：待處理、執行中、已完成及已撤回），亦可透過篩選以彙整未完成項目，並在下一次會議前提供清單，確保每個決議事項都能達到整合，不因工程繁忙而被遺忘。
    
    本系統的會議紀錄功能，是將營建行政與現場工務深度整合的橋樑。它不僅提升了文書處理效率，更透過數位化的手段，強化了工程專案的風險控管能力與執行力，為專業的工程團隊提供了一個嚴謹、透明且具備溯源價值的管理平台。
