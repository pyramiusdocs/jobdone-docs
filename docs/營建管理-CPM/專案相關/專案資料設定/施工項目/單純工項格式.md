### 02｜編輯施工項目 <a href="#create" id="create"></a>

如需修改現有的施工項目內容，請依循以下步驟執行：

1. **開啟編輯模式：**&#x65BC;施工項目頁面右上方點選 ![](https://2002560121-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FEqUCL3D5WQfpxJw8NL3P%2Fuploads%2FGV8m0FMQlMurVKWFkQeb%2Fimage.png?alt=media\&token=17571675-e0bd-402c-8b69-350eccbcf888) 圖示，系統即會切換至編輯狀態。

![](../../../../images/8d629cd8d567cff5.png)

2. **修改資料：**&#x5728;此模式下，您可以直接針對各工項的「編號」、「項目及說明」、「單位」、「數量與單價」以及「管理標籤」進行即時修正。

![圖五 - 修改工項資料(一)](../../../../images/0fef82ae6701691a.png)

3. **刪除工項：**&#x82E5;該工項已不需施作且尚未有施工日誌回報紀錄，可點選該列最右側的 ![](https://2002560121-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FEqUCL3D5WQfpxJw8NL3P%2Fuploads%2FL24hvOj7OhbClLoTPPMu%2Fimage.png?alt=media\&token=fed9c28a-139d-4ef4-834b-3452b7a4b407) 圖示進行移除。
4. **儲存變更：**&#x4FEE;改完畢後，請務必點選右上方的 ![](https://2002560121-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FEqUCL3D5WQfpxJw8NL3P%2Fuploads%2FCAa8ht67AhQaQQoZl9Tr%2Fimage.png?alt=media\&token=1c01b64c-e0a2-4073-bc95-06753138a9dc) 按鈕，系統將自動依照修正後的數值重新跑算全案的進度權重。

### 03｜增補紀錄 <a href="#create" id="create"></a>

工項建置完畢後，若後續因變更設計、追加項或其他合約變更事宜，您可針對該工項持續回報『增補紀錄』。系統支援單一工項建立多筆增補，每筆紀錄皆須詳實附上三個關鍵數據：增補日期、增補數量、增補單價。

{% tabs %}
{% tab title="增補紀錄對進度計算的實務邏輯" %}
系統對於施工進度(%)的計算並非採取全案齊頭式修正，而是具備『時間軸』的動態計算：

1. 分段計算邏輯：系統會以『增補日期』作為切割點。在該日期之前的施工日誌進度，仍維持原契約的權重計算；自增補日期當日（含）以後，該工項的進度分母與權重才會根據新的總量（原契約 + 增補量）進行即時變動。
2. 價金權重校正：由於增補的單價可能與原契約不一致，系統會自動將『原契約複價』與『所有增補複價』進行加權計算。這能精確反映該工項在合約總價值中的真實佔比，確保產出的數據具備法律與財務上的嚴謹度。
{% endtab %}

{% tab title="對進度曲線(S-Curve)的影響" %}
當您填寫增補紀錄後，整體的預定與實際進度曲線將會產生動態位移：

1. **曲線下降趨勢：**&#x7531;於該次增補之數量追加，導致分母（總價）變大，在已施作量不變的情況下，累計進度百分比（%）會出現****合理下降****。
2. **權重重新分配：**&#x589E;補後的 S-Curve 將自動依據新的價金權重重新分布。免除了人工手動修正進度表與權重分配的繁瑣與誤差。


!!! danger
    關於施工項目中設定的契約量、單價以及後續的「增補紀錄」，其對累積進度與 S-Curve 的動態影響，僅會在 『標準版施工日誌』 中呈現（因精簡版日誌無進度計算）。

{% endtab %}
{% endtabs %}

當專案發生變更或追加項時，請依據以下步驟在系統中反映數據異動：

1. 開啟編輯模式(圖七)：於施工項目頁面右上方點選 ![](https://2002560121-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FEqUCL3D5WQfpxJw8NL3P%2Fuploads%2FnpGEZpld2DLXbiCrW9xa%2Fimage.png?alt=media\&token=1a2fb920-5a85-4eb7-bbed-ecffa7cd5ba5) 圖示，進入編輯狀態。

![](../../../../images/f0b8f3c1c9886755.png)

2. 新增紀錄(圖八)：找到欲填寫增補的工項，在其『增補』欄位內點選 ![](https://docs.jobdone.cc/user_guide/~gitbook/image?url=https%3A%2F%2F2002560121-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252FEqUCL3D5WQfpxJw8NL3P%252Fuploads%252Fcuz3KPKsvgvU6EFieNKb%252Fimage.png%3Falt%3Dmedia%26token%3D04b6d2d7-e591-4842-b173-7448aa0968e6\&width=300\&dpr=3\&quality=100\&sign=d96d9b54\&sv=2)。
3. 填寫數據：在彈出的視窗中，填寫增補日期、增補數量、單價等資訊。若該工項涉及多階段變更，可持續點選 ![](https://2002560121-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FEqUCL3D5WQfpxJw8NL3P%2Fuploads%2FaLlI60cqMh85LFiL9amr%2Fimage.png?alt=media\&token=606b6d9d-cb78-49e8-9529-ebccfd1f71a5) 以新增多筆紀錄。
4. 確認與儲存：確認所有增補資訊填寫無誤後，務必點選畫面右上方的 ![](https://docs.jobdone.cc/user_guide/~gitbook/image?url=https%3A%2F%2F2002560121-files.gitbook.io%2F%7E%2Ffiles%2Fv0%2Fb%2Fgitbook-x-prod.appspot.com%2Fo%2Fspaces%252FEqUCL3D5WQfpxJw8NL3P%252Fuploads%252FQZAubmKBkvwoI7Kd75tw%252Fimage.png%3Falt%3Dmedia%26token%3D8a501534-d35c-494f-9e34-0d48f844ef14\&width=300\&dpr=3\&quality=100\&sign=9e238351\&sv=2) 按鈕。


!!! info
    #### 重要規範與實務提醒
    
    * **施工日誌即時連動：**&#x8ACB;務必注意，增補紀錄一旦儲存，系統將立即重新計算進度分母。這會直接影響到施工日誌端呈現的『累計完成量』與『進度百分比（%）』，建議在取得核定公文或正式變更單後再行登錄。
    * **資料的可修正性：**&#x70BA;因應營建實務中可能發生的單價調整或核定數量差異，日後若有異動，一樣可以回來修正已儲存的增補紀錄。系統會根據修正後的數值，自動校正該工項在 S-Curve 中的價金權重。
    * **增補日期生效規則：**&#x518D;次提醒，增補對進度的影響是從『增補日期』當天（含）開始生效。若您需要追溯先前的進度權重，請確認日期填寫的準確性，以確保專案歷史進度軌跡的真實性。


![圖八 - 填寫增補紀錄](../../../../images/ad1a28540e330c21.png)
