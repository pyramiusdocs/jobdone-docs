# 文件中心


!!! warning
    文件中心功能僅能&#x65BC;**「網頁版」**&#x4F7F;用。


營建專案時程動輒數年，涉及的圖紙、公文、合約及計畫書成千上萬。在過去，這些資料常散落在個人通訊軟體、工務所電腦或雜亂的紙本卷宗內。當現場施作與圖說版本出現落差，或因查不到最新的「施工計畫」而盲目施工時，產生的拆改成本與工期延誤往往難以估計。

Jobdone 的『文件中心」』是為了建立專案的單一事實來源。可將營建流程中所有關鍵文件，從合約標單、近期預定計畫、各專業分工施工圖 (Shop Drawings) 到材料檢驗報告 (Mill Certificates)，全部整合在結構化的管理架構下。這不僅是儲存檔案，更是確保從辦公室端的專案經理到現場的工地主任，手上的資訊永遠同步。

透過這套系統，工程人員不再需要耗費大量時間在群組中翻找舊訊息，或擔心誤用過時圖紙。每一份上傳的計畫變更或材料歷程，都會成為專案履約的鐵證，在日後面對竣工驗收、工程追加減，甚至是保固維修與責任追溯時，提供最完整且具備法律效力的數位資產依據。

***

系統支援依據營建實務流程建立大型資料夾叢集，讓多層級管理符合工務邏輯，而非單純的檔案堆放（如：

> * **合約與商務：**&#x7D71;一收納主合約、變更設計公文、分包商承攬書及估驗請款紀錄，方便財務與公務核對。
> * **技術與圖說：**&#x5340;分建築、結構、機電等各專案 Shop Drawings，並針對『施工計畫書』與『計算書』再設立資料夾專區。
> * **材料與品質：**&#x5C08;門存放各項材料的出廠證明、試驗報告及進場檢驗紀錄，解決竣工結案時資料缺漏的痛點。
> * **進度與計畫：**&#x7BA1;控每週/每月動態調整的施工計畫表與近期預定計畫，確保現場班組作業與傯進度掛鉤。

<kbd>**資料夾的『審核准駁』機制**</kbd>

為了確保上傳文件的正確性，管理員在新建資料夾（或資料夾叢集）時，可針對該目錄指定特定的『審核者』。此機制建立了檔案正式生效前的緩衝防線，具體運作邏輯如下：

{% tabs %}
{% tab title="草稿區隔離審查" %}
若該資料夾設有審核者，使用者上傳檔案後，檔案不會立即出現於正式目錄，而是先行移動至『草稿區』進行暫存。
{% endtab %}

{% tab title="准駁機制與正式生效" %}
審核者會收到系統通知，並在草稿區近行文件內容確認。僅有在審核者執行『審核通過』後，該檔案才會正式發佈至資料夾中並供相關人員檢視；若審核不通過，則該檔案即從草稿區退回，確保正式環境中每一份環境的嚴謹性。
{% endtab %}
{% endtabs %}


!!! info
    #### 實務情境
    
    * **施工計畫與進度同步：**&#x91DD;對動態調整的施工計畫表，上傳後即時更新、確保各分包分組（如鋼筋、模板、混凝土）的進度銜接無誤。
    * **材料證明與驗收配套：**&#x9032;場材料的出廠證明、試驗報告與自主檢查紀錄可即時歸檔，大幅縮短竣工驗收時翻找紙本檔案的時間，並在孤雁請款時作為附件依據。
    * **變更設計與責任釐清：**&#x7576;設計變更發生時，透過文件中心將變更前後的圖說、簽核公文完整留存，作為日後追加減價與釐清施工責任的重要憑據。
