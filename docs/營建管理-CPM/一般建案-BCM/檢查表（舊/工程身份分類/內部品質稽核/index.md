# 內部品質稽核

本檢查表以系統化、獨立化之查驗，已判定品質活動與相關結果是否符合預定計畫，及這些計畫是否有效被執行，並能適切達到品質目標。

檢查表讓您充分落實品質制度稽核：

* [x] **對於工地之各項計畫書、施工要領、施工圖表、品質管理標準、自主檢查等，是否落實執行**。
* [x] **由作業文件及紀錄確認執行工作者確實依據作業流程執行。**
* [x] **回饋機制之有效性。**

***

## 系統檢查表功能

### **進行現場稽核作業  ➙  品質控管紀錄與抽查**

系統提供多種檢查時機，包含施工前、施工中、施工後、檢驗停留點等。

監理等相關人員可使用 Jobdone App 現場建立欲使用之檢查表，並檢查是否有品質缺失項目，直接於檢查表填寫檢查結果。

包括：

**「 合格 」、「 不合格 」、「 不適用 」、「 未填寫 」** 等。           &#x20;


!!! info
    **「 不適用 」：**&#x7576;現場檢查項目不符合檢查表中相關規定時，選擇此選項。



!!! warning
    **請注意：**
    
    
    
    1. **先行建立專案檢查範本**
    
    由於進行檢查表設置時需要使用 **「** [**專案檢查範本**](../zhuan-an-jian-cha-fan-ben) **」**，因此在進行檢查之前，請務必確認已經充分建立並設置好相關的檢查範本。這樣可以確保檢查流程順利進行，並避免因範本未設置而導致的錯誤或延誤。
    
    
    
    2. **確保 「 協力廠商 / 外部聯絡人 」 資料已建立完畢**
    
    當檢查項目 **「 涉及外部廠商或協力廠商 」** 時，需在新增檢查時填寫相關的 **「 責任廠商 」**。                     不過，這一欄為 **「 非必填 」** 項，若該檢查項目由自身廠商負責或由多個廠商負責，則可不填寫。     若該檢查項目由自身廠商負責或由多個廠商共同負責，則可在針對不合格項目 **「 發出改善單 」** 時，逐一填寫 **「 責任廠商 」** 與 **「 缺失改善人 」（即受文者）**。
    
    
    
    3. **確保 「 施工圖面 」 與 「 建地結構 」 等資料已妥善建立**
    
    開始進行現場檢查時，針對各檢查項目您將依需求選用先前設立好的相關資料，並將選定的檢查項目應用於建築物、結構或其他設施的具體施工位置。也可利用已設定好的 **「 施工圖面 」** 在檢查過程中使用該圖面來清晰標示出建地結構圖上的檢查位置，明確標示出結構圖上的檢查位置。  這樣不僅能確保實際施工位置與設計圖一致，還能幫助檢查人員準確無誤地完成檢查紀錄。

### **品質缺失改善 ➙ 提出改善行動通知書**

針對缺失項目可發出 **「 改善單 」**，選定缺失改善人並限期改善。

缺失人於期限內改善完畢，可針對 **「 改善前 」、「 改善中 」、「 改善後 」** 詳細記錄改善過程，亦需填寫相關原因分析、矯正措施等。

審查相關廠商的缺失改善行動是否完備，由此建立完整稽核後流程與追蹤，包括：

**調查行動：**&#x78BA;認缺失產生原因與影響範圍  **➙**  系統提供 **「 根本原因分析 RCA 」** &#x20;

**改善行動：**&#x4F9D;調查結果進行改善  **➙**  系統提供 **「 我的改善單 」** 讓相關人員進行改善回報&#x20;

**矯正行動：**&#x9632;止缺失再度產生的因應措施。

***

### **追蹤各改善行動**

所有專案內部改善單將於 「 專案改善單 」 頁面羅列出，包含改善狀態、主旨、受文者、改善期限。                       系統亦會於改善期限即將到期、改善預期的個案等給予醒目提示。讓您充分追蹤改善進度。

若改善人未於規定之期限內完成，系統提供 **「 會議記錄 」**，使監理等稽核人員召開主管會議（工地會議）   將情況充分提報，直至缺失改善完成。
