#### 01 - 1｜拍照存入相簿

在使用 App 影音日誌功能進行現場勘察時，針對拍攝照片的儲存邏輯，系統提供了高度的隱私與空間管理彈性。您可以根據個人的使用習慣或手機容量狀況，透過系統設定自行決定照片的流向。

當您使用 Jobdone 內建相機功能或『專案浮水印相機』（標註時間、座標、專案資訊）進行拍攝時，可於設定中切換以下模式：

{% tab title="一併存入手機相簿（同步儲存）" %}

> **實務效益：**&#x65B9;便您在離線狀態下，或是透過通訊軟體（如 Line）快速分享照片給協力廠商，且即便日後刪除雲端紀錄，手機仍保有原始檔備份。

> **實務效益：**&#x71DF;建工程照片動輒數千張，此模式能有效避免個人相簿被大量施工照片（如鋼筋、泥作、垃圾清理等）佔據，保持個人手機空間的純淨，並減緩手機容量爆滿的可能性。

#### 01 - 2｜圖像上傳壓縮比例

您可以根據案場的網路環境（如地下室收訊較弱）調整上傳品質。選擇較高的壓縮比可大幅提升上傳速度並節省流量；若需保留極高精細度（如裂縫觀測），則可調低壓縮比以維持影像原真。

值得一提的是，即使在設定中選擇了『效能』模式（最大化壓縮），Jobdone 的圖像處理引擎仍能保持極佳的視覺辨識度。

### 03｜暫存媒體庫

在營建現場，穩定的網路環境往往是種奢求。為了因應地下室、偏遠山區或強電磁干擾等離線情境，與網頁端直接手動上傳至媒體庫的邏輯不同，App 端的媒體庫是為了『現場搶拍』與『延時同步』而設計的。

**📸 App 離線觸發機制：暫存媒體庫的運作邏輯**

此功能機制的核心在於確保『紀錄不因斷網而中斷』，讓現場工程師能先補捉畫面，帶環境允許時再進行歸檔。


!!! info
    #### 💡 實務現場管理效益
    
    * 現場人員不再需要為了等照片轉圈上傳而停在原地。在斷網的區域中，您可以連續拍攝數十張鋼筋配置圖，所有照片都會乖乖待在 App 的暫存隊列中，等您回到收訊良好的地方再一鍵上傳。
    * 傳統手機相機拍照容易與系統紀錄脫節，導致是候補日誌時『對不上照片』。透過 Jobdone 離線相機觸發並匯入媒體庫，於後續填寫影音日誌時能輕鬆選取並引用。當您處於收訊不佳的現場時，可直接開啟 App 內提供的離線浮水印相機進行拍照。


{% tabs %}
{% tab title="離線拍照觸發" %}
此時拍攝的照片會帶有精確的浮水印資訊（可自行調整，如經緯度、時間），並暫時儲存在 App 的離線資料暫存區。
{% endtab %}

{% tab title="手動上傳與媒體庫匯流" %}
當您回到辦公室或網路微服後，您需要進入 App 內的離線紀錄區執行『手動上傳』。

這些離線拍攝的照片再上傳成功後，會自動流向雲端的『暫存媒體庫』。

* **後續引用：**&#x6B64;時，無論您是想在 App 端補做日誌，或是回到辦公室透過網頁端進行精細編輯，都能在暫存媒體庫中看到這些『離線搶拍』的成果。
{% endtab %}
{% endtabs %}

### 04｜離線相機

承接前段關於暫存媒體庫的機制，接下來詳細介紹支撐這套邏輯的核心工具：『離線相機』。

在營建工程的實務環境中，網路訊號往往極不穩定，從深開挖的地下層道高聳的電梯井，訊號斷斷續續的常態。離線相機的設計初衷，就是為了打破『有網路才能做紀錄』的限制，確保現場工程師的快門節奏不被收訊環境干擾。


!!! warning
    #### ⚠️ 重要限制說明：影音日誌的離線機制
    
    在使用影音日誌離線功能時，請務必區分其與其他功能（如檢查表的離線紀錄）的邏輯差異，影音日誌的核心在於『影像搶拍』，而非全資料的自動帶入：
    
    * **僅支援『圖片』離線存儲：**&#x5F71;音日誌的離線功能主要服務於****離線拍照****。當您在收訊不良處使用離線相機時，一桶鎖定的是現場的影像與浮水印資訊。
    * **標題與描述需『另行建立』：**&#x4E0D;同於檢查表可同時填寫離線紀錄（說明、缺失等），影音日誌的紀錄標題、文字描述、相關分項工程、建地結構......等，無法在離線狀態下填寫。
    * **操作邏輯建議：**&#x73FE;場人員的標準動作應為：在離線環境下優先完成拍照，待網路恢復、照片上傳至暫存媒體庫後，再回到紀錄頁面點選 ![](https://2002560121-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FEqUCL3D5WQfpxJw8NL3P%2Fuploads%2FXJAmnZeRzHVd1jnvdTOo%2Fimage.png?alt=media\&token=b9474323-85c8-4792-a1bc-38a9f857842d)，手動補齊日誌的文字標題及內容，再引用對應的圖片。

{% tabs %}
{% tab title="精確的時間戳記" %}
鎖定快門落下的瞬間，確保紀錄的真實性。
{% endtab %}

{% tab title="空間座標資訊" %}
透過 GPS 離線紀錄定位資訊。
{% endtab %}

{% tab title="專案預設標籤" %}
如專案名稱、施工地點等。讓照片在進入媒體庫前就已具備專業身份。

完成離線照片的編輯與標記後，系統會將這些紀錄安全地存放於『離線暫存區』。為了確保數據不遺漏，App 介面提供了直觀的提示機制，引導您完成最後的上傳步驟：

**📤 離線檔案同步流程**

1. 紅點提示機制(圖四)：請留意 App 影音日誌畫面右下方的 ![](https://2002560121-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FEqUCL3D5WQfpxJw8NL3P%2Fuploads%2FYQYcHwOcObeoGpveKp55%2Fimage.png?alt=media\&token=217955d8-73da-4a3a-a688-16bc081149e8) 圖示。若該圖示出現，即代表目前暫存區內尚有未上傳的離線圖檔。這是一個重要的防呆提醒，確保您在離開案場前不會忘記同步資料。
2. 查看未上傳清單(圖五)：點選 ![](https://2002560121-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FEqUCL3D5WQfpxJw8NL3P%2Fuploads%2FtfG7kzXPFziHwDQ8Tznz%2Fimage.png?alt=media\&token=c19041bc-6671-4b14-8b2c-b88e7b8b5c87) 圖示，系統將列出所有待命中的離線檔案。在此介面中，您可以預覽即將上傳的照片，確認無誤。
3. 啟動檔案上傳：確認網路環境穩定後（建議連接 Wi-Fi 或處於 4G/5G 收訊良好處），點選最下方的 ![](https://2002560121-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FEqUCL3D5WQfpxJw8NL3P%2Fuploads%2FHTTPcgYLJUhdA9HmaO82%2Fimage.png?alt=media\&token=eb77b7a8-b149-42da-9216-0068f0b05297) 按鈕，系統將開始批次將檔案匯流至雲端暫存媒體庫。
