# 紀錄列表

## 01｜紀錄列表

進入「影音日誌」功能後，您可點選頁籤切換至<kbd>**紀錄列表**</kbd>分頁，此處彙整了專案內所有的影音檔案與施工紀錄。在列表頁中，您可以快速閱覽各項紀錄的分類與進度，點擊任一項目即可進入編輯模式，針對影像標籤或內容細節進行修正；若需建立新的現場紀錄，點選<kbd><mark style="color:purple;">**+新增紀錄**<mark style="color:purple;"></kbd>即可開始上傳影音並填寫對應的結構化資訊。

![紀錄列表模式](../../../../images/4833530925ff0362.png)

### 01 - 1｜暫存媒體庫

營建案場常面臨地下室或電梯井網路收訊不佳的情境，導致紀錄無法即時送出。本系統提供「暫存媒體庫」功能，作為手機 App 與網頁 Web 端的數據中繼站，確保每一份珍貴的影像都不因網路波動而遺失。

{% tabs %}
{% tab title="離線上傳機制（App 端）" %}
當施工現場網路收訊不良時，您可使用「離線上傳」功能，將拍攝的圖片與影片先行儲存於 App 內的離線區。待回到工務所或網路流暢環境後，系統將提示您進行同步，屆時可將所有檔案一併上傳至「暫存媒體庫」。
{% endtab %}

{% tab title="雲端同步與預處理（Web 端）" %}
暫存媒體庫支援跨平台同步。除了 App 端的離線回傳，行政人員或管理層亦可直接在網頁端將大量拍攝的檔案預先上傳至媒體庫中。

1. 您可以先將大量影像上傳至暫存區，待有空暇時，再從媒體庫選取照片來建立正式的影音日誌，並補齊建地結構、協力廠商等標籤。
2. 檔案進入暫存媒體庫即代表已受雲端保護，不佔用手機儲存空間，且網頁與 App 資訊即時連動，方便辦公室與現場協同作業。
{% endtab %}
{% endtabs %}

### 01 - 2｜新增紀錄

如圖十三，進入紀錄列表頁面後，點選右上角之<kbd><mark style="color:purple;">**+新增紀錄**<mark style="color:purple;"></kbd>即可進入新增畫面。

在新增紀錄時，您需填寫日期、標題、備註並上傳相關的影音檔案。為了確保日誌具備後續檢索與統計的價值，請務必選用該筆紀錄對應之建地結構、施工項目、分項工程、相關材料及相關協力廠商等結構化標籤，完成後點選儲存即可建立完整的施工履歷。

![](../../../../images/7ecf11c7069707ae.png)

進入「新增紀錄」畫面後，系統將引導您填寫結構化的施工紀錄。這些資訊並非單純的文字，而是未來進行進度核銷、品質追蹤及責任釐清的關鍵依據。

**（一）基礎資訊填寫**

{% tabs %}
{% tab title="日期(Date)" %}
系統預設為當日，但可依實際施工時間調整。
{% endtab %}

{% tab title="標題(Title)" %}
建議採「位置 + 工種 + 動作」命名（例如：A棟 1F 區段 鋼筋綁紮查驗）。清晰的標題能讓團隊成員在紀錄列表中不需點開檔案，即可快速掌握現場概況。
{% endtab %}

{% tab title="備註(Remarks)" %}
用於詳細記錄現場狀況。建議包含：施工精度數據（如間距、水平）、氣候影響、或需後續追蹤的異常點。這是在影像之外，最重要的補充證言。
{% endtab %}
{% endtabs %}

**（二）結構化標籤（依據專案設定自動帶入）**

以下欄位均連動專案後台之預設資料，確保全案場資訊的一致性：

{% tabs %}
{% tab title="建地結構(Location)" %}
將影像精確定位於建物部位（如：B1F、頂板、A棟梯廳）。這在未來發生爭議或修繕需求時，能秒速過濾出該位置的所有歷史影像。

有關建地結構之編列說明，請參閱 ➙ [construction\_structure](../../../project_level/project_data/construction_structure "mention")
{% endtab %}

{% tab title="施工項目(Item)" %}
對應具體的作業內容（如：連續壁澆置、防水打底、鋼筋組立）。此標籤有助於分析各項工序的執行週期。

有關施工項目之編列說明，請參閱 ➙ [construction\_item](../../../project_level/project_data/construction_item "mention")
{% endtab %}

{% tab title="分項工程(Section)" %}
歸類於合約中的工程大項（如：土建、機電、裝修）。方便管理層從大範圍視角檢視各專業領域的施工歷程。

有關分項工程之編列說明，請參閱 ➙ [zhuan-an-fen-xiang-gong-cheng](../../../project_level/project_data/zhuan-an-fen-xiang-gong-cheng "mention")
{% endtab %}

{% tab title="材料(Materials)" %}
紀錄該工項所使用的關鍵物料（如：4000psi 混凝土、D19 鋼筋、防護漆料）。建立影像與材料履歷的連結，確保施作與報審規格吻合。

有關材料管理之相關說明，請參閱 ➙ [material\_management](../../../project_level/project_data/material_management "mention")
{% endtab %}

{% tab title="協力廠商(Vendor)" %}
標註負責的包商名稱。這不僅是進度追蹤，更是責任劃分與履約評價的重要數據。

有關協力廠商之編列說明，請參閱 ➙ [subcontractor](../../../project_level/project_stakeholders/subcontractor "mention")
{% endtab %}
{% endtabs %}

**（三）**&#x65B0;增日誌時，系統提供兩種檔案載入模式，您可以根據現場環境與作業習慣選擇最合適的方式：

{% tabs %}
{% tab title="從暫存媒體庫選取" %}
若您先前已透過 App 離線同步，或是在網頁端預先上傳了大量照片，可點選此選項。系統會開啟媒體庫視窗，讓您直接勾選已在雲端緩衝區的圖檔，快速轉化為正式紀錄。此模式適合處理批次圖檔或已完成標註的編輯檔。


!!! warning
    #### ⚠️ 重要提示
    
    由於暫存媒體庫目前專為圖檔管理設計，若欲上傳影片紀錄，則務必使用「直接上傳檔案」模式。

{% endtab %}

{% tab title="直接上傳檔案" %}
若您手邊已有整理好的檔案（存於電腦或手機相簿中），可選擇此模式直接將檔案匯入日誌。這能跳過媒體庫的中轉步驟，適合單筆、即時性的紀錄。
{% endtab %}
{% endtabs %}

{% tab title="建立人(Author)" %}

顯示該筆日誌（包含所有照片與影片）所使用的雲端空間大小。這對於大型專案的檔案管理與備份規劃提供了量化的參考數據。

### 01 - 3｜篩選紀錄

考量當案場執行時間拉長，累積的影音紀錄將動輒數千筆。系統提供『篩選功能』，您無需在漫長的列表中逐一翻找，即可精確定位特定的施工關鍵點。

如圖十四、十五，點選列表右上方的 ![](https://2002560121-files.gitbook.io/~/files/v0/b/gitbook-x-prod.appspot.com/o/spaces%2FEqUCL3D5WQfpxJw8NL3P%2Fuploads%2FosWjpMUGSX7y4LmBEWYc%2Fimage.png?alt=media\&token=221ae4d9-8860-49f6-b2d8-337808877b1b) 圖示，即可展開篩選選單。系統提供多維度的搜尋條件，讓您可以根據現場需求進行交叉檢索：
