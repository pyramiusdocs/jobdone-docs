# 執行檢查工作

## 01｜填寫檢查紀錄

當檢查任務建立完成後，現場人員即可隨時透過行動裝置展開即時查驗，將現場施作狀況轉化為數位紀錄：

如圖一，於檢查工作頁面，點選欲執行之檢查工作，即可進入該檢查內部，查看所有檢查項目並填寫檢查結果。


!!! warning
    系統預設會自動勾&#x9078;**『僅顯示我的工作』**，優先呈現由您發起或負責的任務。若您需要查閱全專案負責的項目或專案全局進度，請務必手動關閉此選項，以免無法查看完整的檢查清單。


**檢查結果說明**

在執行現場查驗時，針對每一項檢查條目，系統提供三種明確的判定結果。這三種狀態將決定後續的報表呈現與流程走向：

{% tabs %}
{% tab title="合格(Pass)" %}
代表該施工項目符合技術規範與設計圖說。
{% endtab %}

{% tab title="不合格(Fail)" %}
代表該項目存在品質缺失，須進行整改。
{% endtab %}

{% tab title="不適用(Not Applicable)" %}
代表該項目在本次查驗範圍或當前施工階段中****不需執行****。

* **實務情境：**&#x82E5;範本包含『電梯井檢查』，但當前工區並無電梯工程；或範本包含『雨天施工防護』，但查驗當日為晴天，即可勾選不適用。
* **報表呈現：**&#x52FE;選不適用的項目在產出正式報表時，會明確標註為『不適用』，確保報告邏輯完整，證明該項目並非『漏檢』而是『不需檢查』。
{% endtab %}
{% endtabs %}

    #### ⚠️ 重要操作導覽：區分「檢查紀錄」與「結果判定」
    
    在執行查驗任務時，請務必辨識正確的點擊區域，以確保進入正確的操作介面：
    
    請特別注意（如圖二所示）：若欲針對項目進行『合格/不合格』的結果判定，請直接點擊該項目下方之框框。請勿點擊右側的『＋』符號，因為該符號是導向『檢查紀錄』視窗（用於新增細部描述），而非判定檢查結果。

### 01 - 1｜檢查紀錄

在執行查驗作業時，系統會根據您的判定結果（合格/不合格）啟動不同的引導流程，確保品質數據能完整留存並具備可追蹤性。

{% tabs %}
{% tab title="結果判定為合格" %}
系統視該項目符合品質標準，為提升作業效率，系統不會主動彈出視窗（檢查紀錄）。然而，若該項目涉及重要數據（如：量測值、壓力數值）或需留存完工照，您仍可主動於右側點選『＋』圖示進行補充，填寫包含檢查位置、標準、描述、圖面標示、附圖及附件等資訊。
{% endtab %}

{% tab title="結果判定為不合格" %}
系統將視為品質缺失，並自動彈出『檢查紀錄』填寫視窗。除了檢查位置、檢查標準及描述等外，此時必須詳細登載缺失細節，以啟動後續的改善追蹤與複驗流程。如：缺失責任人與改善人： 明確指派負責整改的人員，落實責任歸屬。


!!! info
    建議透過『圖面標示』定位缺失點，並上傳『附圖（缺失照片）』與相關『附件』作為改善前的依據。&#x20;

{% endtab %}
{% endtabs %}

在營建現場，單一檢查項目（標準）往往需對應多個實體構件。系統支援「每筆檢查項目皆可建立多筆檢查紀錄」，這能完美對應複雜的施工查驗情境：

> **實務範例：連續壁工程 — 壁體鋼筋籠及預埋件檢查**
>
> 假設本次查驗項目是「預埋連接器（Coupler）位置與數量」，而現場當日要施作的是「一整個單元（Unit）」，內部包含多組預埋件。
>
> * 操作方式： 您不需要為每一組預埋件建立一份檢查表，只需在同一個「預埋連接器」項目下，利用『檢查紀錄』 功能（點擊右側的「＋」）一一回報。
> * 數據呈現： 您可以針對「單元左側」、「單元右側」、「轉角處」等不同位置分別新增紀錄，並在每一筆紀錄中上傳對應的照片、填寫實測數值。
> * 混合判定（核心價值）： 若其中 5 處預埋件合格、1 處偏移不合格，合格的紀錄會作為品質存證存檔，而不合格的那一筆紀錄則會自動標記缺失，並同步至「缺失檢查項目總表」，確保每一處細節都能被追蹤到結案。

**【視窗欄位說明與實務應用】**

{% tabs %}

{% tab title="檢查標準" %}
顯示該項目的技術規範。執行人員可在此比對實測數據，例如標準為「垂直度 ± 3mm」，可於描述欄填寫實測結果，落實數據化管理。
{% endtab %}

{% tab title="描述" %}
用於記錄現場執行細節。

> 範例：「經現場實測，鋼筋間距誤差控制在 5mm 內，且箍筋彎鉤角度均符合 135 度要求，無鬆脫現象。」
{% endtab %}

{% tab title="附圖" %}

App 系統支援自動帶入時間、座標、工程名稱、施工位置之浮水印，確保照片的真實性，符合營建業品管規範（如公共工程委員會之照片紀錄要求）。
{% endtab %}

{% tab title="附件" %}
可上傳與該紀錄相關的電子文件，如：材料出廠證明、試驗報告 PDF、或施工說明書截圖。這讓查驗紀錄不僅是照片，更具備完整的法律證據效力。
{% endtab %}
{% endtabs %}

### 01 - 2｜更多功能

在執行檢查工作時，App 提供多項「快捷功能」與「彈性管理」工具，協助您應對現場多變的狀況，確保查驗流程不中斷。包含：<kbd><mark style="color:purple;">**一鍵不適用**<mark style="color:purple;"></kbd>、<kbd><mark style="color:purple;">**一鍵未填寫**<mark style="color:purple;"></kbd>、<kbd><mark style="color:purple;">**一鍵發起改善單**<mark style="color:purple;"></kbd>、<kbd><mark style="color:purple;">**新增項目與管理**<mark style="color:purple;"></kbd>

在營建現場，為了應對突發的施作異動或單一工序的特殊性，系統提供了快速同步所有項目狀態的功能：

當原定的檢查表項目與當日檢查工作完全不相符，或依使用者自身情況而定時，系統提供『一鍵不適用』及『一鍵未填寫』，讓您能夠一次更改所有檢查項目的檢查結果。

{% tabs %}
{% tab title="一鍵不適用(Apply All N/A)" %}
當整份範本中的項目在當前階段完全不需執行時（例如：原定進行封板檢查，但現場因故停工或工序調整），可使用此功能快速將所有『未填寫』項目標記為『不適用』。這不僅節省逐一勾選的時間，更能確保該次檢查紀錄在產出報告時具備完整的合理解釋。
{% endtab %}

{% tab title="一鍵未填寫" %}
若需要重新評估或誤選了大量項目的結果，可使用此功能將所有已選為『不適用』的判定一併清空，恢復至初始狀態重新進行判定，確保數據錄入的正確性。
{% endtab %}

{% tab title="一鍵發起改善單" %}
查驗結束後，發現多個項目標註為「不合格」。

系統會自動彙整所有不合格紀錄，一次性產生缺失改善通知單並派發給改善人，無需針對所有缺失重複填寫改善單資訊。
{% endtab %}

{% tab title="新增項目與管理" %}
支援現場增列範本外的特殊查驗內容，確保「計畫外」的施工細節也能被數位化追蹤。您可以自定義名稱與標準，並隨時調整其在檢查表中的排序。
{% endtab %}
{% endtabs %}

#### **01 - 2 - 2｜一鍵發起改善單**

當檢查表完成後，若存在多個「不合格」項目，您可以使用此功能快速啟動整改流程，無需逐項建立改善單。


!!! warning
    #### 請注意
    
    若該檢查表已開啟複驗流程，則 <kbd>**一鍵發起改善單**</kbd> 功能僅限於 **複驗** 階段使用。此設計是為了確保改善單能精確對應複驗後的最終判定結果，避免流程衝突。


**1. 自動化資訊帶入機制**

由於此功能為批次發送，為了確保溝通效率，系統會以檢查紀錄自動帶入：

{% tabs %}
{% tab title="主旨與描述" %}
系統將自動帶入該檢查項目的項目的『檢查題目』作為主旨，並將您在檢查紀錄中填寫的文字描述直接轉為缺失描述。
{% endtab %}

{% tab title="影像佐證" %}
該項目對應的所有附圖（包含手繪標註）會自動夾帶於改善單中，確保收件人能精確對應缺失位置。
{% endtab %}
{% endtabs %}

**2. 受文者派發規則 (Recipient Logic)**

您可以根據專案管理的需求，選擇以下兩種派發邏輯：

{% tabs %}
{% tab title="優先發送給缺失改善人(Priority Delivery)" %}
**邏輯：**&#x7CFB;統會優先檢查每一筆紀錄是否有指定特定的『缺失改善人』。

**結果：**&#x6709;指定對象的紀錄將發送給該人員；若該筆紀錄為指定，則會發送給您在批次畫面中所選擇的『統一受文者』。

**適用情境：** 當同一張檢查表涉及多個不同專業廠商（如水電、泥作、木作）時使用。
{% endtab %}

{% tab title="統一發送給受文者(Unified Delivery)" %}
**邏輯：**&#x7CFB;統將忽略個別紀錄的指定人選。

**適用情境：**&#x7576;所有缺失皆歸屬於同一承包商，或需統一由該廠商的工地負責人對口處理時使用。
{% endtab %}

!!! warning

    2. **防止遺漏缺失：** 所有的查驗需求應在初驗時一次界定完畢。若現場臨時有新增的查驗需求，應另行建立一筆新的「檢查工作」，而非在現有的複驗流程中安插新項目。
    3. **合規與稽核：** 在外部稽核（如三級品管）時，紀錄的「一致性」是檢查重點。鎖定複驗清單能證明管理過程嚴謹，未隨意竄改或事後補件。

## 02｜附圖功能說明

**操作導覽：靈活運用附圖功能 (Image Attachment Methods)**

在執行查驗並填寫檢查紀錄時，完善的影像佐證是品質管理的核心。系統提供兩種靈活的上傳方式，滿足不同的現場作業情境：

{% tabs %}
{% tab title="現場即時紀錄：選擇相簿/直接拍照" %}
* **操作：**&#x9EDE;選後可直接啟動手機相機進行即時拍攝，或從手機相簿中選取已存有的照片。
* **適用情境：**&#x9069;用於發現突發缺失、或是需要針對特定查驗細節進行近距離特寫時。
{% endtab %}

{% tab title="系統資源調度：暫存媒體庫" %}
* **操作：**&#x9EDE;選後可直接從系統內部的媒體空間選取照片。
* ****來源說明：****此處可選取的照片，是同步自該專案中****『影音日誌內的暫存媒體庫』****。
* **適用情境：**
  * 避免重複拍攝： 若工程師在巡視現場時，已先透過「影音日誌」大量拍攝了當日的施工狀況，在進入「檢查表」時，只需從暫存媒體庫中勾選對應照片即可，無需重新開相機拍照。

{% endtab %}

## 03｜離線上傳功能

在營建現場（如地下室、連續壁溝槽或隧道工程），網路訊號往往不穩定。系統內建了強大的『離線紀錄模式』，確保您的查驗進度不會因斷網而中斷。此模式會透過以下兩種情境自動或手動觸發：

## 04｜送交審核

確認所有檢查項目皆已填寫完畢，且紀錄確實無誤後，即可點選<kbd><mark style="background-color:blue;">**初驗完成送審**<mark style="background-color:blue;"></kbd>。系統將依據該檢查所使用之『檢查流程』，將該筆紀錄即時傳送給相關管理人員（如：工地主任或專案管理人員）進行評估與簽認。


!!! warning
    #### 請注意
    
    若仍有檢查項目處於『未填寫』之狀態，則系統將限制提交，無法將紀錄送往下一階段審核。


<div>![圖一 - 尚未完成檢查](../../../../../../images/7e5e3ac77a6b0304.png) ![圖二 - 無法提交](../../../../../../images/cc467d231711d8e6.png) ![](../../../../../../images/bd0c94c9b4f3bbb1.png)</div>

（一）將檢查送審後，該檢查工作之狀態即會由<kbd><mark style="color:blue;">**待檢查**<mark style="color:blue;"></kbd>轉為<kbd><mark style="color:yellow;">**初驗審核中**<mark style="color:yellow;"></kbd>。

1. 狀態轉為<kbd><mark style="color:yellow;">**初驗審核中**<mark style="color:yellow;"></kbd>後，原執行人員將無法再隨意修改檢查結果或紀錄內容。這確保了審核者所看到的數據與現場提交時是一致的，維持紀錄的嚴謹性與不可篡改性。
2. 此時，審核人員（如工地主任或專案管理人員）的待辦清單中會出現該筆任務，並查看您上傳的所有照片、圖面標示及實測數據。並獲得核定、退回的權限。

（二）在審核過程中，若審核人員（如工地主任 或 專案管理人員）認為檢查紀錄不夠詳盡、照片不清晰或判定結果有誤，可執行****退回****操作：

若審核人員將檢查紀錄退回，該檢查工作之狀態則會由<kbd><mark style="color:yellow;">**初驗審核中**<mark style="color:yellow;"></kbd>更改為<kbd><mark style="color:blue;">**修正中**<mark style="color:blue;"></kbd>。

1. 當狀態轉為<kbd><mark style="color:blue;">**修正中**<mark style="color:blue;"></kbd>時，系統會解除資料鎖定，重新開放編輯權限給原執行人員。執行人員可針對審核意見進行內容補正、更換照片或重新判定。
2. 執行人員修正完畢後，需再次點選「送交審核」，狀態將重新回到審核中，直到審核人員核定通過為止。
