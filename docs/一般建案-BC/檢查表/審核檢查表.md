# 審核檢查表

檢查表送審後，待審核的檢查表會顯示 「 審核中 」 狀態，擁有專案經理權限的人可對檢查表進行審核，結果包含 「 [退回修改](#refuse) 」 及 「 [通過並簽名](#sign) 」 。

## 網頁版

### 退回修改 <a href="#refuse" id="refuse"></a>

點選 「 退回修改 」 後，檢查表狀態將改為 「修正中」 並退回給檢查人員，允許檢查人員修正檢查表內容。

![](images/b9c6ef6ea7b30d46.png)

![](images/d09d5451025a796d.png)

### 通過並簽名 <a href="#sign" id="sign"></a>

當檢查表內容無誤時，可點選 「 通過並簽名 」 進入[**複驗**](re-web)階段。若不需要複驗，可在 「 **檢查表設定** 」 中停用此功能。


!!! warning
    按下 「 通過並簽名 」 進入複驗後，檢查表就無法進行任何改動，也無法退回審核階段，在結案前請確認內容無誤。


![](images/5176be0088246cc8.png)

## APP

進入專案內的檢查表後，選擇 「 審核中 」 分頁即可看見待審核的檢查表。

審核完畢後點選下方的 「 審核通過 」 或 「 退回 」 即可完成審核。
